## Хорошо

- **Чуть быстрее** — на 17% меньше wall-time, на 30% меньше tokens.
  Видимо системник стал чуть короче или модель меньше «рефлексирует».
- **Сразу пошла в код** — без stalling на архитектурных вопросах.
  Можно интерпретировать как «модель смелее».

## Плохо

- **Та же фундаментальная проблема** — нет tool-loop, нет SR
  применения. Это не косметика, это **архитектурное недостающее
  звено**, и v0.4 его не привнесла.
- **Сразу пошла в код БЕЗ уточнений** — на v0.3 модель спрашивала
  «формат JSON? argparse?». На v0.4 — нет. Если это **благодаря**
  изменению системника на v0.4 — наблюдение. Если **случайность**
  одного прогона — не учитываем. Без повторов не отличить.

## Гипотезы о причинах

- v0.4 — это release-cut **без архитектурных изменений** в
  agent/SR/tools пайплайне. См. git log v0.3.0..v0.4.0 — там
  скорее всего cosmetic.

## Архитектурный smell-check

v0.4 — это **второй frame baseline**. Видим что одна-единственная
точка не достаточна — нужно 2 для подтверждения паттерна.

L1». Следующий скачок ожидаем где-то на v0.5+ когда landed
code_with_retry с SR-парсингом.

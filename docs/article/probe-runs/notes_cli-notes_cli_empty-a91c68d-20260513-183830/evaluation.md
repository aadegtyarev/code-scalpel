# Evaluation: notes_cli × v0.5.0

## Reached level: **L3** (= v0.3-v0.4)

`code_with_retry` инфра landed, но **в системнике всё ещё нет
`_CODE_MODE_ADDENDUM`** — модель не учится формату патчей.

## One-liner

v0.5: L3 как baseline. `code_with_retry` функция появилась (commit a06e0c9), но **без mode-инструкции** модель не использует её для патчей. Самый дорогой прогон по compute — 13 requests / 79k tokens с 0 done.

## Главный пересмотр

**Раньше**: «code_with_retry landed но модель не научена SR» —
неточно.
**Точно**: на v0.5 системник всё ещё содержит только
`_PLAN_MODE_ADDENDUM`. **Code mode не учит** модель ни SR, ни
write_file (его и нет на v0.5). Поэтому появление
`code_with_retry` как функции в коде — это **инфраструктура для
будущего**, не работающий ход здесь и сейчас.

## Архитектурный smell-check

`code_with_retry` появился как функция, но **без addendum'а в
системнике** модель не понимает что в code mode другие правила.
Это **архитектурный полу-шаг**: infra v0.5, prompt-обвязка
позднее.

Подтверждает урок главы 6 девлога: **обогащай данные, не
промпт** — но также и обратное: **инфра без правильной
prompt-обвязки бесполезна**. v0.5 — наглядное доказательство.

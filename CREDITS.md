# Благодарности

Проект code-scalpel опирается на чужие идеи и инструменты. Этот файл — про
тех, кому стоит сказать спасибо. Никакой код **не копировался** один в один —
это благодарность за подсмотренные подходы и за полезные инструменты.

## Идеи

- **[aider](https://github.com/Aider-AI/aider)** (Paul Gauthier, лицензия
  Apache 2.0 — совместима с нашей AGPL-3.0) — донор формата SEARCH/REPLACE
  блоков для редактирования файлов. Изначально мы использовали unified diff,
  но он плохо работает со слабыми локальными моделями: они путают счётчики
  `@@`, выдают противоречивый контекст, ломают форматирование. aider давно
  решил ту же задачу и пришёл к формату без счётчиков строк. Мы повторили
  его спецификацию в `patch/edit_block.py` — собственная реализация парсера
  и аппликатора, идея формата и каскад стратегий «exact → whitespace-tolerant
  → leading-blank-strip» взяты у них. Код **не копировался** — все строки в
  `edit_block.py` написаны с нуля.

## Инструменты

- **[Claude Code](https://claude.com/claude-code)** (Anthropic) — IDE-агент,
  с помощью которого пишется этот проект. Архитектура TUI (вход внизу, чат
  растёт снизу вверх, слэш-команды с автокомплитом) сложилась через диалог
  с ним.

- **[Textual](https://textual.textualize.io/)** (Textualize) — TUI-фреймворк.
  Без него UI пришлось бы делать на curses вручную.

- **[textual-autocomplete](https://github.com/darrenburns/textual-autocomplete)**
  (Darren Burns) — выпадающий автокомплит для слэш-команд.

- **[LM Studio](https://lmstudio.ai/)** — локальный сервер для qwen и других
  open-source моделей с OpenAI-совместимым API.

- **[qwen2.5-coder](https://github.com/QwenLM/Qwen2.5-Coder)** (Alibaba) —
  основная модель v0.1. На ней проверялась гипотеза «слабая локальная модель
  может стабильно править код, если давать ей маленький контекст и
  валидировать формат».

- **[unidiff](https://github.com/matiasb/python-unidiff)** — парсер unified
  diff в Python, использовался в ранней версии аппликатора.

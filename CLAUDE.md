# code-scalpel

TUI coding agent для работы с кодом через слабые локальные LLM.
Принцип: маленький контекст, маленький patch, быстрый тест, контролируемая автономность.

## Ключевые файлы

| Файл | Назначение |
|---|---|
| `docs/plan.md` | Полная архитектура и роадмап. Источник правды. |
| `docs/article_draft.md` | Техническая статья о процессе проектирования. Пишется параллельно с кодом. |

## Dev flow

```bash
# Установка (один раз)
python -m venv .venv && source .venv/bin/activate
pip install -e ".[dev]"

# Перед коммитом
ruff check . && ruff format --check .
mypy code_scalpel/

# Тесты
pytest                        # все тесты
pytest -x                     # стоп на первом падении
pytest --cov --cov-report=term-missing

# Автофикс
ruff check --fix . && ruff format .
```

Правила:
- `ruff format` — форматтер (вместо black). Запускать перед коммитом.
- `mypy --strict` — нарушения блокируют коммит.
- Все публичные функции и методы должны иметь аннотации типов.
- Тесты в `tests/`, мокируют LLM и shell через `tests/mocks.py`.

**Тесты пишутся вместе с кодом, не после.** Каждый коммит с новым модулем должен включать тест.
Нет теста — нет коммита.

**Перед коммитом обязательно:**
1. `ruff check . && ruff format --check .` — без ошибок
2. `mypy code_scalpel/` — без ошибок
3. `pytest -x` — все тесты зелёные

## Как работать с планом

`docs/plan.md` — живой документ. При реализации:
- Если что-то в плане не так — **сначала поправь план, потом пиши код**
- Если принято архитектурное решение которого нет в плане — добавь
- Не добавляй фичи сверх того что описано в текущей версии роадмапа

Роадмап: раздел 31. Текущая цель — **v0.3 hooks** (см. список с галочками).

будущим сессиям):

```
  whole / window / find. Глава 18 статьи.
```

## Тестовая модель

qwen2.5-coder-14b-instruct в LM Studio (`http://localhost:1234/v1`).
LM Studio должна быть запущена при тестировании агента.

Альтернативы по результатам кросс-модельного бенча (см.
`docs/bench-models.md` — 7 моделей, 24 теста):
- **gemma-4-26b-a4b** — 100% качество, в 2.5× медленнее
  (если важно максимальное качество, готов терпеть latency)
- **qwen3.5-9b** — 79% качество, fallback для low-RAM
- coder-14b остаётся дефолтом потому что Pareto-оптимал —
  лучший по скорости + качеству одновременно.

## Стиль кода

- Python 3.11+, async везде где есть I/O
- DI через конструктор, `app.py` — единственный composition root
- Никаких магических чисел — всё в конфиге (`config.py` + `pydantic`)
- Комментарии только когда WHY неочевиден
- Тесты через `MockLLMAdapter` и `MockShellRunner` из `tests/mocks.py`
